        // We intentionally do NOT break out of the loop after the capture:
        // unlike raylib's synchronous `TakeScreenshot`, bgfx's clean exit

        // presented (and, for `--screenshot`, queued the capture for) this

        // not `N + screenshot delay`); only the EXIT is held open while a
        // screenshot is still pending (`screenshot_req != null`, so `--ticks`
        // can't cut off a capture whose `--after` delay hasn't elapsed) or
        // mid-flush (so the async bgfx readback can flush to disk first).
        // Without that gate, `--ticks` + `--screenshot` together would exit
        // before the file lands.

        }
